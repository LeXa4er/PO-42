## Введение

Проект представляет собой систему просмотра товаров отдела классической музыки музыкального магазина, разработанную на языке программирования C# с использованием Windows Forms в качестве пользовательского интерфейса и Microsoft SQL Server для хранения данных.

## Функциональные требования

1. **Аутентификация**
   - Администраторы приложения должны иметь возможность авторизоваться в системе, используя свои учётные данные.

2. **Просмотр информации о товарах**
   - Просмотр информации о товарах с функциями фильтрации и поиска;
   - Возможность выбора типа носителя;
   - Возможность фильтрации по наличию товара в отделе.

3. **Административный функционал**
   - Система регистрации учётных записей для администраторов с сохранением информации в базе данных;
   - Управление учётными записями администраторов приложения;
   - Добавление информации о новых товарах;
   - Просмотр и редактирование информации о товарах.

## Технические требования

- Использование языка программирования C#;
- Использование Windows Forms для разработки графического интерфейса;
- Использование Microsoft SQL Server для хранения информации в базе данных;
- Корректная обработка возникающих в процессе работы программы ошибок и исключений;
- Интуитивно понятный пользовательский интерфейс.

## Дополнительные требования

- Проект должен быть разделен на логические слои (пользовательский интерфейс, работа с базой данных).
- Исходный код проекта должен быть понятно задокументирован и структурирован.

# Описание форм и их функций

## 1. Главная форма

- **Описание:** Форма, предназначенная для просмотра товаров, которые могут быть доступны в данном отделе магазина.

- **Функции:**
  - Отображение списка товаров с учётом	поиска по их наименованию, введённых фильтров и наличия в магазине;
  - Вывод подробной информации о выбранном товаре (конкретная пластинка или диск).

## 2. Форма авторизации

- **Описание:** Форма, предназначенная для аутентификации администраторов приложения.

- **Функции:**
  - Ввод данных учётной записи (логин и пароль);
  - Проверка введённых данных на соответствие с информацией в базе данных;
  - Переход к административной части приложения при успешной аутентификации;
  - Уведомление об ошибке при неудачной аутентификации.

## 3. Формы добавления информации

- **Описание:** Формы, предназначенные для добавления в таблицы базы данных информации о новых товарах и администраторах.

- **Функции:**
  - Ввод необходимых данных (зависят от таблицы, в которую будет сохранена информация);
  - Подтверждение корректности введённых данных и уведомление об обратном в противном случае;
  - Уведомление об ошибке при неудачном сохранении информации.

## 4. Формы редактирования информации

- **Описание:** Формы, предназначенные для редактирования информации о новых товарах и администраторах в таблицах базы данных .

- **Функции:**
  - Отображение всей информации, хранящейся в конкретной таблице;
  - Возможности добавления, удаления и редактирования информации, хранящейся в конкретной таблице;
  - Уведомление об ошибке при неудачном сохранении изменений.

## 5. Форма с информацией о программе

- **Описание:** Форма, предназначенная для отображения информации о приложении.

- **Функции:**
  - Предоставление информации о названии, версии и авторах данного программного обеспечения.

# Дополнительные Требования

## Обработка переменных

- **Понятные имена переменных:** Переменные должны иметь осмысленные имена, описывающие их назначение, что улучшает читаемость кода.

- **Инициализация переменных:** Все переменные должны быть корректно инициализированы перед их использованием во избежание ошибок, связанных с неопределёнными значениями.

- **Константы:** Используйте константные переменные для значений, которые не должны изменяться в ходе работы программы. Это уменьшит вероятность возникновения ошибок.

## Использование конструкции `try-catch`

- **Корректная обработка ошибок:** Используйте конструкцию `try-catch` для обработки исключений и ошибок. Все блоки кода, которые могут вызвать исключения, должны быть заключены в блок `try-catch` для предотвращения сбоев приложения.

- **Логгирование ошибок:** При возникновении ошибок выводите сообщения для упрощения диагностики и дальнейшего устранения возникших проблем.

- **Информативность сообщений об ошибках:** Предоставляйте подробные сообщения об ошибках, которые помогут администраторам понять причины возникшей проблемы.
